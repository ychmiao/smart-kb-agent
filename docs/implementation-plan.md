# 分阶段实现计划

本计划以“每个阶段都可验证、主链路逐步闭环”为原则。下一阶段开始前，应先确认上一阶段的编译、测试和文档均通过。

## 阶段 0：规范与目录骨架（已完成）

- 建立 `AGENTS.md`、README 初版和单体分包目录。
- 明确技术栈、代码规范、项目边界与禁止事项。
- 验收：仓库结构与 `docs/project-prompt.md` 一致，未引入业务实现。

## 阶段 1：工程基线与本地基础设施

- 创建 `pom.xml`、应用入口、`application.yml` 和环境变量模板。
- 创建 Docker Compose：MySQL、Redis、etcd、Milvus、MinIO、Attu。
- 创建 `sql/init.sql`，落实六张业务表及必要索引。
- 建立统一 `Result<T>`、错误码、全局异常处理和基础日志规范。
- 验收：`mvn clean verify` 通过；容器健康；应用可启动并生成 OpenAPI 文档。

## 阶段 2：认证与知识库管理

- 实现注册、登录、Token 刷新、BCrypt、JWT 拦截器和 `UserContext`。
- 实现知识库创建、列表、删除及资源归属校验。
- 创建并删除 `kb_{kbId}` Milvus collection，明确跨存储失败补偿策略。
- 验收：认证接口、Token 生命周期、越权访问和知识库 CRUD 测试通过。

## 阶段 3：文档上传与异步入库

- 实现文件格式、MIME 和 50 MB 大小校验，原文件写入 MinIO。
- 建立文档处理记录和命名异步线程池。
- 使用 Apache Tika 提取文本，实现 `SemanticChunker` 的段落/句子边界、500 字上限和 100 字重叠。
- 调用 Embedding 并将 chunk 元数据写入 Milvus，维护完成/失败状态。
- 实现文档列表、删除，以及 MySQL、MinIO、Milvus 的清理与补偿。
- 验收：四类文档均可处理；分块边界测试通过；失败状态可诊断且可安全重试。

## 阶段 4：统一 LLM 网关与容错

- 定义 `LlmGatewayService` 和供应商适配层，支持同步对话、流式对话、Embedding。
- Chat 按 DeepSeek → Qwen 路由，Embedding 固定使用 Qwen。
- 分别配置超时和 Resilience4j CircuitBreaker；全部失败时抛出统一异常。
- 异步记录模型、调用类型、延迟、成功状态和脱敏错误信息。
- 验收：使用 Mock Server 覆盖成功、超时、熔断、降级、双供应商失败和流式中断。

## 阶段 5：查询理解与向量检索

- 实现 `QueryRewriteService`，使用最近 2 轮上下文判断检索意图并改写问题。
- 严格解析模型 JSON；解析失败时回退到“需要检索 + 原始问题”。
- 实现 `RetrievalService`：生成查询向量、Milvus TopK 检索、批量补全文档名。
- 增加分数阈值和空结果策略，但首版不增加 Rerank。
- 验收：闲聊、完整问题、指代问题、非法 JSON、空召回和越权知识库测试通过。

## 阶段 6：多轮会话与 SSE RAG 闭环

- 实现会话创建、历史查询和最近 10 条 Redis 上下文缓存。
- 编排查询重写、可选检索、RAG/闲聊 Prompt 和流式模型调用。
- 按 `rewrite → token → sources → done` 输出 SSE，并定义稳定错误事件。
- 异步持久化用户消息、AI 回答、引用、重写结果和模型信息；更新 Redis。
- 验收：首轮/多轮、无需检索、模型降级、客户端断开和持久化失败场景测试通过。

## 阶段 7：联调、质量与交付

- 增加单元测试、基础设施集成测试和核心 API 冒烟测试。
- 校验并发上传、幂等性、事务边界、异步异常、资源归属和日志脱敏。
- 完善 OpenAPI、Docker 健康检查、启动脚本、配置说明和故障排查。
- 更新 README 为真实可运行状态，补充演示流程与面试讲解材料。
- 验收：全新环境按 README 可一次启动；完整演示“注册 → 建库 → 上传 → 入库 → 流式问答”。
